Datum: 2026-07-12

| Thema | Erkenntnis |
|-------|------------|
| Wrapper-Klassen | Man kann vorhandene `List`-Methoden in eigenen Methoden einer Wrapper-Klasse delegieren (z. B. `isEmpty()` ruft `languages.isEmpty()` auf) |
| Boolean-Rückgabe | `if (bedingung) return true; else return false;` ist identisch zu `return bedingung;` |
| List-Methoden | `isEmpty()`, `add()`, `remove()`, `get(0)`, `size()`, `contains()` sind die grundlegenden `List`-Methoden der Java Collections API |
| IDE-Nutzung | Im Editor (ZED) direkt Methoden einer Klasse aufrufen/autovervollständigen, um zu sehen, welche Methoden verfügbar sind und wie sie funktionieren |
